Связь заказа и исполнителя

Обновление

/api_v2/order/review/{id}/ [PATCH]

Для обновления необъодимо использовать id, полученный при просмотре заказа по ендпоинту /api_v2/order/{id}/ [GET]

Json Example

{
  ...
  "customer_score": 0,
  "order_in_order_review": [
    {
      "id": 0, <------- Этот id.
      "status": "looked",
      "to_price": 0,
      "to_deadline": "2022-01-18T20:33:23.903Z",
      "cancel_reason": "low_budget",
      "show": true
    }
  ],
  "term": 0
}

Статусы исполнителя в заказе:

  • looked
  • reconciliation
  • rejected
  • accepted
  • in_progress
  • done

Причины причины отмены заказа:

  • low_budget
  • short_time
  • incomprehensible_task
  • not_enough_skills
  • Im_busy

Json Example

{
  "status": "looked",
  "cancel_reason": "low_budget",
}
